Chapter 3: System Design

The design chapter provides the blueprint of the C.U.S.T FYP Management Portal, bridging the gap between requirements and implementation. It outlines how different components of the system are structured, interact with each other, and support the required functionality. This chapter ensures all user requirements are properly addressed and can be effectively translated into development.

3.1 Software Architecture

The system follows a MERN Stack (MongoDB, Express, React, Node.js) architecture, specifically utilizing a Client-Server Architecture.

High-Level Architecture Components:

  1. Frontend (Client Layer): Built with React.js, the frontend handles the User Interface (UI) and User Experience (UX). It communicates with the backend via RESTful APIs using Axios.
  2. Backend (Server Layer): Built with Node.js and Express.js, the server handles business logic, authentication, and data processing.
  3. Database (Data Layer): MongoDB is used as the primary database. Mongoose is used as an Object Data Modeling (ODM) library.
[Architecture Diagram: User <-> React <-> Express <-> MongoDB]

3.2 Data Modeling

The system uses a NoSQL approach, where data is stored in collections. The relationships are managed using ObjectIDs in Mongoose.

Core Entities (NoSQL/JSON Structure)

1. GenUser (Unified User Model)

Stores all system users including Admins, HoDs, Coordinators, Faculty, and Students.

  "department": "ObjectId (Ref: Department)",
  "program": "ObjectId (Ref: Program)",
  "term": "ObjectId (Ref: FYPTerm)",
